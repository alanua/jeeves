# 09 — Ecosystem Watchlist

Last updated: 2026-04-24

## Purpose

This watchlist tracks external information sources and the types of signals that may be useful for Jeeves.

The watchlist is not a source of truth. It is a signal filter.

## Evaluation rule

All ecosystem news must be evaluated by practical project fit:
- Does it affect architecture?
- Does it affect provider strategy?
- Does it affect security or permissions?
- Does it suggest a useful pattern?
- Is the claim verified?

Unverified hype must not drive architecture.

## Signal categories worth monitoring

High-value categories:
- computer-use / browser-use
- MCP and browser integration
- coding-agent security
- OpenClaw risks and patterns
- practical multi-agent DevOps workflows
- N8N / Supabase / MCP automation patterns
- project-scoped contexts and workspaces
- audio-to-context ingestion
- low-code Telegram / Make-style workflows
- wrapper-around-the-model product thesis

## Handling unverified claims

Unverified ecosystem claims should be marked as noise until confirmed.

Example accepted rule:
- claims like “OpenAI bought OpenClaw” are ignored unless officially confirmed

## Useful output format for future news analysis

When analyzing a new post or repo, classify it as:
- useful pattern
- maybe useful later
- provider/model signal
- security signal
- product/use-case signal
- hype/noise
- verified fact
- unverified claim

Then map it to one of the Jeeves layers:
- routing
- orchestration
- agents
- policy/permissions
- memory/handoff
- knowledge-base subsystem
- action layer
- skills layer
- browser-use/computer-use
- provider strategy
- security/release hygiene
