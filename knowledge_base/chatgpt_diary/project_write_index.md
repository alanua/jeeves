# ChatGPT Project Write Index

Status: CONFIRMED_CANON
Scope: index of where ChatGPT records project/task continuity, canon, diary, handoff, and audit notes
Created: 2026-05-03

## Purpose

This file answers: what is written where?

It is the map that a new ChatGPT chat should read after the boot files and diary index to quickly find the right project branch, handoff, diary, audit, or structured facts.

This is not raw memory. It is a navigation/index layer.

## Standing GitHub write permission

Oleksii grants standing permission for ChatGPT to write public-safe docs/KB/handoff/diary/task-file updates to the GitHub KB without asking every time.

Allowed without asking again:

- public-safe canon notes
- startup/boot protocol updates
- diary entries
- handoff entries
- recovery/audit notes
- project write indexes
- runner-readable public-safe task files
- docs-only KB cleanup

Not allowed under this standing permission:

- secrets, API keys, tokens, credentials
- raw banking/finance/health/email/private document contents
- unredacted private client data
- destructive code or infrastructure changes without explicit task context
- publishing private Google Drive contents into public GitHub
